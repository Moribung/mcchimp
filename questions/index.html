How the Question System Works

McChimp games don't have fixed question banks hardcoded into them. Instead, every game loads questions from a modular JSON file — a structured document you can write yourself, generate with AI, or download from our library.

This means you can play any game with any topic. An MMA career with driving theory questions. A football career tested on economics. The game engine doesn't care — it only cares about the structure of the file.

Difficulty Tiers MMA Game

Every question set is organised into four difficulty tiers. The game selects questions based on your current rank and career phase — lower-ranked fights get easier questions, title fights get elite ones.

Easy Medium Hard Elite

The game also tracks your personal score on each question (+1 correct, −1 wrong) and adjusts the effective difficulty dynamically. A hard question you've answered correctly twice becomes medium difficulty for you specifically.

Fighter-Owned Questions MMA Game

Each opponent in the division owns a specific question. You face the same question when you rematch them — until you beat them twice consecutively, at which point they get a fresh one. The more you know, the easier the rematch becomes.

How to prompt an AI to make a question set

Any major AI assistant — ChatGPT, Claude, Gemini — can generate a fully compatible McChimp question set if you give it the right instructions. Download the guide below, paste it into your AI chat, describe your topic, and it will produce a ready-to-use JSON file.

The guide covers the exact schema, field rules, difficulty tiers, ID format, and quantity guidelines. The AI does the rest.

File Format

A question module is a single .json file. You do not need all four tiers — include whichever you have questions for.

{ "name": "Module Name", "description": "One-line description (optional)", "tiers": { "easy": [ ...questions ], "medium": [ ...questions ], "hard": [ ...questions ], "elite": [ ...questions ] } }

Question Object

{ "id": "PREFIX-TI-001", "question": "What is the question text?", "options": ["Option A", "Option B", "Option C", "Option D"], "answers": [0], "explanation": "Shown after the answer is revealed. Optional." }
FieldRequiredRules
idRecommendedStable string, unique within module. Format: ABC-EA-001. Used for score tracking.
questionRequiredPlain text. No length limit but keep it readable.
optionsRequiredArray of 2–7 strings.
answersRequiredArray of zero-based indices into options. At least one.
explanationOptionalShown after the answer is revealed. Good for teaching.

Difficulty Tiers

The tier a question lives in determines when it appears. The game also shifts effective tier at runtime based on your history with each question (+1 correct, −1 wrong) — write each question at its genuine difficulty and let the system handle the rest.

TierWhen it appears
easyEarly career, lower-ranked opponents
mediumMid-career, mid-ranked opponents
hardTop organisations, ranked opponents
eliteTitle fights and championship bouts

Single-Answer vs Multi-Select

Questions can have one or more correct answers. Wrong selections subtract a point — guessing is penalised.

// Single answer "answers": [2] // Multi-select — question text should say "Select all that apply." "answers": [0, 2, 3, 4]

ID Format Convention

PREFIX-TI-NNN — e.g. SCI-HA-007

PartMeaning
PREFIX2–4 uppercase letters for the module (MMA, HIS, SCI)
TIEA (easy) · ME (medium) · HA (hard) · EL (elite)
NNNZero-padded number: 001, 002…

Quantity Guidelines

More questions means less repetition in long careers. A module with fewer than ~40 total will see repeats.
