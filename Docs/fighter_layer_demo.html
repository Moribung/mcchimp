    How this works: Every sprite is drawn by stacking filled rectangles (no curves, no lines) onto a 32×48 canvas
    and displaying at 8× scale with image-rendering: pixelated. Each layer is an independent draw pass keyed to
    a body-part anchor — hair and beard anchor to the head position, shorts and belt anchor to the hips. Switching pose moves
    the anchors; all attached layers follow automatically. The idle bob (1px sine wave) demonstrates this live.
    


    Palette swap works by resolving colour roles (skin, shorts, hair) to actual hex values at draw time —
    the same region list recolours instantly. State-driven layers (org shorts, belt, damage) are controlled by career state,
    not player choice. Identity layers (skin, hair, beard) persist in save_data.
    


    To replace a sprite with PixelLab art: keep the same canvas dimensions and anchor pixel positions,
    drop the PNG into static/sprites/, done. The compositor is file-based and needs no code changes.
